## Key Rules

- The API owns all domain logic and persistence; agents persist nothing.
- Only the `agentclient` module calls the agent service — no other module imports Spring AI.
- Schema migrations live in `src/main/resources/db/migration/` (Flyway); no undocumented manual schema changes.
- Every AI-generated output affecting grading, feedback, or student-facing content requires an `ApprovalEvent` before it is acted on.
- Evidence entities (`AgentExecutionLog`, `ApprovalEvent`, `UsageEvent`, `RevenueEvent`, `CostEvent`) are first-class, not afterthoughts; `RevenueEvent.related_party` is required.
- No student login in MVP: access via signed token links (`AssessmentInvitation`); `LearnerRef` is a minimal reference record, not an account.
- Closed assessment grading is always deterministic against a frozen answer key.
