## Key Rules

- Paquete base: `cl.gradeops.ai.api` — no cambiar
- Layout por feature: `{feature}/{domain,usecase,api,infra}` — ningún archivo de dominio en `infra/` y viceversa
- `domain/`: Java puro — sin `@Component`, `@Service`, `@Entity`, ni imports de Spring/JPA
- `usecase/port/`: interfaces de puertos de salida — implementadas siempre en `infra/`
- `api/`: solo adaptadores HTTP — sin lógica de negocio; mapea request→command, result→response
- `infra/`: entidades JPA, adaptadores de persistencia, clientes externos
- Inyección de dependencias: solo por constructor — sin `@Autowired` en campos
- Sin `@Data` en entidades JPA — usar `@Getter @Setter @Builder`
- Migraciones Flyway: numeración secuencial `VN__`; nunca modificar migraciones ya aplicadas
- Firebase Admin SDK: solo en `infra/firebase/`; nunca expuesto fuera del bounded context `auth`
- `StoragePort` y sus adaptadores viven en `shared/infra/storage/`
- Los orquestadores de use cases inyectan use cases atómicos (interfaces), no ports directamente
